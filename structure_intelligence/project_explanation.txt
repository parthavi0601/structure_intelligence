Structural Intelligence Platform - Architecture Explanation

Think of this entire software system as a highly intelligent brain monitoring a physical bridge. It has senses (the raw data), a subconscious mind that organizes the data (the data pipeline), a conscious mind that makes decisions (the Machine Learning models), a vocal cord to speak (the Backend API), and a face to show results to human engineers (the Frontend Dashboard).

1. "The Subconscious Mind" - The Data Pipeline (Python)
Before the AI can make predictions, the raw sensor data must be cleaned and organized. This happens in the `pipeline` files:
   - Loaders: Fetch raw CSV datasets and format timestamps properly.
   - Transformers: Standardize large numbers alongside tiny numbers so the AI doesn't get confused, and teach the AI about time cycles using Sine/Cosine waves.
   - Feature Engineering: Looks at trends over time. Calculates "rolling averages" and "lag features" to understand what the bridge's strain or vibration looked like hours ago compared to now.

2. "The Conscious Mind" - The AI Models (run_*.py)
Once the data is clean, the machine learning models analyze it:
   - Behavioral Analysis (`run_behavior_analysis.py`): Learns the bridge's normal "personality" and calculates a "Behavioral Shift Index" if the bridge starts acting out of character.
   - Anomaly Detection (`run_anomaly_detection.py`): Acts as a lie detector using an Autoencoder neural network. If it sees new, weird sensor data that it can't memorize, it flags it with an "Anomaly Score".
   - Risk Prediction (`run_risk_prediction.py`): Acts as the fortune teller. Uses a Random Forest Regressor to look at traffic, weather, and the anomaly scores to calculate a 0-100% "Predicted Risk Score" of structural failure.

All this intelligence is saved instantly into lightning-fast `.parquet` files instead of a standard database.

3. "The Vocal Cords" - The FastAPI Backend (api.py)
A lightweight Python server wakes up whenever the frontend asks for data.
   - It reads the `.parquet` files and compresses the data so it doesn't crash the user's browser.
   - It analyzes the AI scores and dynamically generates human-readable conclusions. For example, if the Risk is 80%, it types out: "⚠ CRITICAL ALERT: Immediate intervention is required."

4. "The Face" - The React Frontend (Dashboard & Digital Twin)
Built with React and Tailwind CSS, this is the modern glassmorphism dashboard the human engineers see.
   - Data Dashboards: Fetch AI scores via the backend API and use `Recharts` to draw beautiful, live graphs of Behavioral Shifts and Anomaly Scores. Colors dynamically turn from Neon Green to Neon Red if thresholds are crossed.
   - Digital Twin Simulation: An interactive 3D model of the bridge built with `React Three Fiber`. As you move sliders for Wind, Traffic, and Temperature, the bridge mathematically calculates Euler-Bernoulli beam physics. The virtual bridge visually sways on-screen and turns red under extreme virtual loads, giving engineers a visual simulation alongside the AI numbers.
